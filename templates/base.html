WebTalk Socket
Projeto Acadêmico

Sistema de comunicação distribuída desenvolvido para a disciplina Fundamentos de Redes e Sistemas Distribuídos - UFCSPA Informática Biomédica.

Professor: João Carlos Gluz

Equipe:

Bruno Costa E Silva Giuliani Lopes Caio Foti Pontes Tainá Machado Selayaran
Arquitetura

Cliente-servidor distribuído implementando comunicação em tempo real via WebSocket e APIs REST para demonstrar conceitos de redes e sistemas distribuídos.

Funcionalidades
  • Chat em tempo real com múltiplos usuários
  • Salas públicas e protegidas por senha
  • Compartilhamento de arquivos (PDF, imagens)
  • Visualização inline de imagens
  • Sistema de soft delete para mensagens
  • Painel administrativo de monitoramento
Tecnologias

Backend:

  • Python 3.8+ com Flask
  • Flask-SocketIO para WebSocket
  • SQLite para persistência

Frontend:

  • HTML5, CSS3, JavaScript ES6+
  • Bootstrap 5 responsivo
  • Socket.IO Client
Conceitos Aplicados
  • Protocolo HTTP: APIs REST (GET, POST, DELETE)
  • WebSocket: Comunicação full-duplex em tempo real
  • Threading: Processamento concorrente de conexões
  • State Management: Sincronização cliente-servidor
  • File Handling: Upload/download seguro com validação
Segurança
  • Validação de tipo MIME e assinatura binária
  • Sanitização de entrada e escape de HTML
  • Controle de acesso baseado em salas
  • Limitação de tamanho de arquivo (16MB)
Objetivo

Demonstrar na prática os fundamentos de redes e sistemas distribuídos através de uma aplicação funcional que implementa comunicação cliente-servidor, gerenciamento de estado distribuído e protocolos de rede modernos.
